# 《审讯室》Demo 迭代方向汇总

> 基于 4 个专业角度 Agent（游戏机制、技术实现、角色叙事、UI/UX 视觉）的交叉讨论，形成本 Demo 级迭代方向文档。

---

## 一、当前项目核心状态

**已完成：**
- 纯前端单页应用（原生 JS + HTML + CSS），无后端/无 LLM
- 3 个案件 × 3 阶段，基于关键词匹配的审讯对话系统
- 4 种语气（威压/温和/欺骗/逻辑）+ 情绪值系统（0-100 抗拒度）
- SVG 程序化生成立绘 + Web Speech API 语音输入 + localStorage 存档
- 案件大厅 / 审讯室 / 结算页 / 档案页 完整页面流程

**核心矛盾：** 当前 `goalKeywords` 精确匹配机制创造了一个"隐性标准答案"体系——玩家在表面上"自主判断"，本质上是在猜测开发者预设的关键词。这与创意方案强调的 **"没有标准答案"** 存在根本冲突。

## 二、Demo 级迭代方向总览

### 2.1 方向矩阵

| 维度 | P0（必须做） | P1（建议做） | P2（可延后） |
|------|-------------|-------------|-------------|
| **游戏机制** | 6种犯人性格+性格-语气相克表；情绪反馈即时标签 | 三维隐藏值系统（突破/误导/信任）；审讯笔记面板 | 证据出示系统；第5语气"沉默施压" |
| **技术实现** | 否定检测修复（"我不是主谋"Bug）；同义词扩展表 | 规则优先级权重；对话历史门控；TTS语音合成 | 模糊匹配/编辑距离；非线性分支结局 |
| **角色叙事** | Tone交叉反应规则；情绪阈值事件（拍桌/哭泣） | 口头禅系统；阶段过渡叙事增强 | 新增角色"周牧野"；近miss回复机制 |
| **UI/UX** | 发送时镜头抖动（按语气分级）；语气按钮色彩语义化 | 三轴心理仪表盘（紧张/防御/信任）；判定反馈差异化 | 移动端抽屉化折叠；结案证据链可视化 |

### 2.2 最高优先级共识（4个Agent一致同意）

1. **修复否定检测 Bug**：当前引擎无法识别"我不是主谋"中的否定词，导致误判。这是体验底线问题。
2. **情绪维度从一维扩展到三维**：紧张度 + 防御值 + 信任度，替代单一的抗拒度，让不同犯人"手感"真正差异化。
3. **性格-语气相克系统**：暴躁型对威压反弹、懦弱型对威压崩溃、冷静型对逻辑松动——让策略选择有意义。
4. **发送消息的物理反馈**：不同语气触发不同强度的镜头抖动，打破网页游戏的平面感。

## 三、分维度详细迭代方向

### 3.1 游戏机制迭代

#### P0 | 6种犯人性格与性格-语气相克表

当前3个角色已覆盖傲慢/紧张/冷静，Demo级别建议补齐到6种性格类型：

| 性格类型 | 代表角色 | 威压 | 温和 | 欺骗 | 逻辑 |
|---------|---------|------|------|------|------|
| 傲慢型 | 沈墨 | 反弹 (+mood) | 小幅松动 | 大幅松动 | 小幅松动 |
| 紧张型 | 林婉 | 崩溃 (大幅降低但可能中断) | 大幅松动 | 中等松动 | 小幅松动 |
| 冷静型 | 老K | 无效 | 小幅松动 | 中等松动 | 大幅松动 |
| 暴躁型 | 【新增】 | 激烈反弹 (可能中断审讯) | 小幅松动 | 小幅松动 | 中等松动 |
| 演技型 | 【新增】 | 小幅松动 | 小幅松动 | 大幅反弹 (识破) | 大幅松动 |
| 懦弱型 | 【新增】 | 大幅松动 (但可能屈打成招) | 大幅松动 | 中等松动 | 小幅松动 |

**Demo实现方式**：在 `cases.js` 中为每个角色增加 `personalityMatrix` 字段，定义4种语气对应的基础 moodDelta 系数。

#### P0 | 连续语气惩罚

连续3次以上使用同一种语气，效果 ×0.7。强制玩家换策略，避免"一招鲜"。

#### P1 | 三维隐藏值系统

将当前单一 `mood` 替换为三维模型：

```js
// 替代当前的 mood: 50
psychState: {
  tension: 60,   // 紧张度：越高越慌乱，可能说漏嘴
  defense: 80,   // 防御值：越高越抗拒，越难突破
  trust: 20      // 信任度：越高越愿意配合
}
```

- **目标达成条件**：`defense < 30 && trust > 50`（而非单纯的关键词命中）
- **陷阱判定**：`tension > 80` 时玩家获得的口供可能不可靠（屈打成招）
- **不同性格的三维初始值不同**：沈墨 defense 高 trust 低；林婉 tension 高 defense 低

#### P1 | 审讯笔记面板

在审讯室右侧/底部增加一个可折叠面板，自动记录：
- 已解锁的关键情报
- 嫌疑人的矛盾陈述（自动标记时间线冲突）
- 当前三维心理状态趋势图

#### P2 | 证据出示系统

在输入区增加"出示证据"按钮，玩家可选择已解锁的情报作为"证据"出示：
- 出示正确证据 → 大幅降低 defense
- 出示错误证据 → 嫌疑人抓住漏洞反击
- 这是创意方案中"证据出示时机"三维策略的简化版

### 3.2 技术实现迭代

#### P0 | 否定检测修复

当前引擎的致命 Bug：玩家输入"我不是主谋"时，`matchRule` 会匹配到关键词"主谋"而误判。

**修复方案**：在 `engine.js` 的 `matchRule` 函数中增加否定词检测：

```js
function matchRule(text, keywords) {
  var t = normalizeText(text);
  var negationWords = ['不', '没', '无', '非', '没有', '不是', '并未'];
  var hasNegation = negationWords.some(function(w) { return t.includes(w); });
  
  return keywords.some(function(kw) {
    var matched = t.includes(normalizeText(kw));
    if (matched && hasNegation) return false; // 否定句不匹配
    return matched;
  });
}
```

#### P0 | 同义词扩展表

当前规则中同义词需手动重复（如"主谋"/"策划"/"组织者"），维护成本高。

**方案**：在 `cases.js` 顶部定义同义词簇：

```js
window.SYNONYMS = {
  '主谋': ['主谋', '策划', '组织者', '指使', '幕后'],
  '当晚': ['当晚', '那晚', '案发当晚', '那天晚上'],
  // ...
};
```

引擎匹配时自动展开。

#### P1 | 规则优先级权重

当前规则按数组顺序匹配，首个命中即返回，无法处理更精确的规则被排在后面导致无法触发的问题。

**方案**：为每条 `responseRule` 增加 `priority` 字段（默认 0），匹配时按优先级排序，高优先级规则优先触发。

#### P1 | 对话历史门控

当前引擎完全不接收对话历史，嫌疑人无法说出有记忆感的台词。

**Demo级简化方案**：在 `generateReply` 中传入 `dialogue` 数组，增加 `historyRequired` 字段：

```js
{
  tone: 'friendly',
  matchKeywords: ['女儿'],
  historyRequired: ['拳馆'], // 必须先聊过"拳馆"才能触发
  reply: '（眼神软化）拳馆的事你都知道了......那我女儿的事，也没什么好瞒的了。',
  moodDelta: -25
}
```

#### P1 | TTS 语音合成

使用浏览器原生 `speechSynthesis` 为嫌疑人回复"配音"，沉浸感提升最显著。

```js
function speakPrisonerReply(text) {
  var utter = new SpeechSynthesisUtterance(text);
  utter.lang = 'zh-CN';
  utter.rate = 0.9;
  utter.pitch = 0.8; // 低沉声音，营造审讯氛围
  window.speechSynthesis.speak(utter);
}
```

#### P2 | 模糊匹配（ROI 低，可延后）

中文场景下编辑距离/模糊匹配的 ROI 较低，因为关键词匹配本身已能覆盖大部分场景。如果做，建议使用简单的拼音首字母匹配作为补充。

### 3.3 角色叙事迭代

#### P0 | Tone交叉反应规则

当前规则只考虑"正确语气"的反馈，错误语气的反馈全部落入 fallback，导致角色表现扁平。

**方案**：为每种性格增加"错误语气"的专属反应：

- 沈墨（傲慢型）+ 温和语气 → "（冷笑）装什么好人？"
- 林婉（紧张型）+ 威压语气 → "（崩溃大哭）不要！我说！我什么都说！"
- 老K（冷静型）+ 欺骗语气 → "（推了推眼镜）警官，你的信息来源不太可靠。"

每条交叉反应约 2-3 条，总计新增约 30-40 条规则。

#### P0 | 情绪阈值事件

当 mood 骤降超过 20 点时，触发一次"情绪爆发"动画+特殊回复：

| 角色 | 阈值事件 | 视觉表现 |
|------|---------|---------|
| 沈墨 | mood > 80 时威压 → 拍桌 | SVG 立绘双手抬起 + 屏幕震动 |
| 林婉 | mood 骤降 → 崩溃 | SVG 立绘眼泪 + 对话字体颤抖 |
| 老K | mood < 30 时 → 摘眼镜 | SVG 立绘眼镜移除 + 眼神变化 |

#### P1 | 口头禅系统

在 fallback 回复池中为每个角色插入口头禅：

- 沈墨："老子..."、"江湖上..."
- 林婉："我...我真的..."、"求你了..."
- 老K："（停顿）"、"警官，这话得有证据。"

#### P1 | 阶段过渡叙事增强

当前阶段过渡只有简单的 `sceneIntro` 系统旁白。建议增加：

- 时间跳跃感（"【审讯室 · 第二天 10:02】"已很好，可增加天气变化、灯光变化）
- 嫌疑人姿态变化（"沈墨双手被铐在铁椅上，眼神比昨天更阴郁"）
- 环境音效提示（"走廊传来脚步声"）

#### P2 | 新增角色提案：周牧野

**角色定位**：演技型 + 高智商型复合角色，填补当前 Demo 的角色空白。

- **表面身份**：广告公司创意总监，社交媒体大V
- **罪名**：职务侵占 + 洗钱 2600 万
- **核心差异**：他会主动"配合"审讯，但说的全是半真半假的剧本
- **玩家挑战**：不是"让他开口"，而是"分辨哪句是真的"
- **犯罪动机**：不是为钱，而是"证明系统可以被愚弄"的智力优越感
- **3阶段目标**：
  1. 让他承认"账目有问题"（他会用大量专业术语混淆）
  2. 找出虚假陈述中的真实线索
  3. 让他承认"是故意的"而非"失误"

#### P2 | 近miss回复机制

当玩家输入的关键词接近但未精确命中时，给出"擦边回复"而非沉默：

```js
// 玩家输入"你组织的吧"
// goalKeywords 包含"组织者"但不包含"组织的"
// 近miss回复：
"（眯眼）组织的？警官，措辞要准确。我是'参与'了，但'组织'？这帽子扣得有点大。"
```

实现方式：为每条规则增加 `nearMissKeywords` 字段。

### 3.4 UI/UX 视觉迭代

#### P0 | 发送消息时的镜头抖动

在 `sendPlayerMessage()` 触发时，为审讯室容器添加短暂的关键帧动画：

```css
@keyframes send-shake-light {
  0%, 100% { transform: translate(0, 0); }
  25% { transform: translate(-1px, 1px); }
  50% { transform: translate(1px, -1px); }
  75% { transform: translate(-1px, -1px); }
}
@keyframes send-shake-heavy {
  0%, 100% { transform: translate(0, 0); }
  20% { transform: translate(-3px, 2px); }
  40% { transform: translate(3px, -2px); }
  60% { transform: translate(-2px, 3px); }
  80% { transform: translate(2px, -3px); }
}
```

**差异化逻辑**：
- 威压语气 → `shake-heavy`（剧烈抖动，模拟拍桌子）
- 其他语气 → `shake-light`（轻微抖动）
- 判定错误时 → 配合 `flash-red` 同步触发剧烈抖动

#### P0 | 语气按钮色彩语义化

当前4种语气按钮选中后共享同一套"琥珀色发光"，缺乏差异化。

| 语气 | 主色 | 选中态增强 |
|------|------|-----------|
| 威压 | 深红 `#A4262C` | 红色边框 + 按钮震动 0.15s |
| 温和 | 暖绿 `#4F6A4B` | 绿色边框 + 轻微呼吸缩放 |
| 欺骗 | 暗紫 `#6B4C7F` | 紫色边框 + 按钮轻微倾斜 |
| 逻辑 | 冰蓝 `#5C7A89` | 蓝色边框 + 网格底纹 |

#### P1 | 三轴心理仪表盘

将当前单一的 `.mood-meter` 重构为纯 CSS 三轴仪表盘：

```css
.psych-gauge {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap: 8px;
}
.gauge-tension .gauge-fill { background: var(--red); }    /* 紧张度 */
.gauge-defense .gauge-fill { background: var(--blue); }    /* 防御值 */
.gauge-trust   .gauge-fill { background: var(--green); }   /* 信任度 */
```

数据层若暂未支持三维，可先做单轴映射：
- `mood = 75` → `tension=60, defense=80, trust=20`

#### P1 | 判定反馈差异化

- **判定正确（Green）**：屏幕中央快速闪过"扫描线"（老电视横向亮线从上到下），隐喻"真相大白"
- **判定错误（Red）**：屏幕四边出现"裂纹"效果（CSS `clip-path`），隐喻"局面崩坏"

#### P1 | 对话流关键情报高亮

当 `result.unlocked` 触发时，触发解锁的玩家消息追加金色边框脉冲动画：

```css
@keyframes clue-reveal {
  0%   { box-shadow: 0 0 0 0 rgba(212,175,55,0.5); border-color: var(--amber); }
  70%  { box-shadow: 0 0 0 10px rgba(212,175,55,0); }
  100% { box-shadow: 0 0 0 0 rgba(212,175,55,0); }
}
```

#### P1 | SVG 立绘 Richer 表情

| 情绪状态 | 新增视觉表现 |
|---------|-------------|
| 紧张度 > 70 | 额头汗珠（SVG circle + opacity 渐变）、瞳孔缩小 |
| 防御值 > 70 | 双臂交叉 SVG 路径叠加、嘴角下撇加剧 |
| 信任度 > 60 | 肩膀放松（身体椭圆 ry 增大）、眼神柔和 |
| 判定错误瞬间 | 立绘整体 brightness(1.3) contrast(1.2) 闪烁 0.2s |

#### P2 | 移动端嫌疑人抽屉化

当前 768px 以下嫌疑人面板被压扁且隐藏情绪条。改为：
- 默认状态：底部一条"嫌疑人信息条"（头像 48px + 名字 + 阶段），高度 64px
- 点击后：从底部弹出抽屉，展开完整三轴仪表盘和已解锁情报
- 发送消息时：抽屉自动收起

#### P2 | 结案证据链可视化

结算页增加极简的"证据链"时间轴：

```
阶段1 ——●—— 阶段2 ——●—— 阶段3
[主谋]    [同伙]    [赃款]
   ✓         ✓         ✓
```

已完成节点为金色实心圆，未完成/误判的为红色空心圆带裂纹。节点间虚线连接，完成时变为实线并播放金色流光。

## 六、4个Agent核心观点摘要

### 游戏机制Agent
> "当前 Demo 的 `goalKeywords` 精确匹配机制，实际上创造了一个'隐性标准答案'体系。最关键的单点改动：将目标判定从'关键词命中'重构为'三维隐藏值累积'（突破值 + 误导值 + 信任值）。"

### 技术实现Agent
> "用'更聪明的规则 + 更丰富的感知反馈'模拟智能，而非追求真正的智能。当前引擎最大的 Bug 是无否定检测。TTS 语音合成可立即为嫌疑人'配音'，沉浸感提升最显著。"

### 角色叙事Agent
