# 🚀 云端部署指南

本项目是纯前端静态应用（无后端依赖），可以一键部署到任意静态托管平台。

---

## 📦 支持的部署平台

### 1. Vercel（推荐）
**优点**: 零配置、一键部署、免费额度充足、全球 CDN

#### 部署步骤：
1. 打开 [Vercel Dashboard](https://vercel.com/dashboard)
2. 点击 **Add New Project** → **Import Git Repository**
3. 选择你的 GitHub/GitLab 仓库（需要先将代码推送到 Git 仓库）
4. **Root Directory**: 留空（使用项目根目录）
5. **Build Command**: 留空（静态文件无需构建）
6. **Output Directory**: 留空
7. 点击 **Deploy**

等待约 1-2 分钟，应用就会部署完成并获得一个 `*.vercel.app` 域名。

---

### 2. Cloudflare Pages（免费）
**优点**: 永久免费、全球 CDN、DDoS 防护

#### 部署步骤：
1. 打开 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 进入 **Pages** → **Create a project** → **Connect to Git**
3. 选择你的 Git 仓库
4. **Build settings**:
   - **Framework preset**: `None`
   - **Build command**: 留空
   - **Build output directory**: 留空
5. 点击 **Save and Deploy**

部署完成后会获得一个 `*.pages.dev` 域名。

---

### 3. Netlify
**优点**: 免费额度充足、直观易用

#### 部署步骤：
1. 打开 [Netlify Dashboard](https://app.netlify.com/)
2. 点击 **Add new site** → **Import an existing project**
3. 连接到你的 Git 仓库
4. **Build settings**:
   - **Base directory**: 留空
   - **Build command**: 留空
   - **Publish directory**: 留空
5. 点击 **Deploy site**

---

### 4. 传统服务器部署（Nginx/Apache）

#### Nginx 配置示例：
```nginx
server {
    listen 80;
    server_name your-domain.com;
    root /var/www/interrogation-room;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(html|css|js|json)$ {
        expires 1d;
        add_header Cache-Control "public, max-age=86400";
    }
}
```

## ⚙️ 部署配置说明

### SPA 路由配置
由于这是单页应用（SPA），需要配置路由回退到 `index.html`：
- **Vercel**: 已配置 `vercel.json`
- **Cloudflare/Netlify**: 已配置 `_redirects`
- **Nginx**: 需要配置 `try_files $uri $uri/ /index.html;`

### 注意事项
1. **无需构建**: 所有文件都是静态资源，直接上传即可
2. **无需后端**: 无 Node.js/Python 依赖，纯浏览器运行
3. **无数据库**: 进度保存在浏览器 LocalStorage
4. **HTTPS**: 部署平台默认提供 HTTPS，语音识别功能需要 HTTPS
