# 《审讯室》Demo 卖点与特色补充建议

> 基于 5 个专业角度 Agent（产品差异化、Demo评审策略、玩法创新、叙事深度、商业化）的交叉讨论汇总。

---

## 一、当前功能完备度评估

**结论：基础框架扎实，Demo级别已达标。但有 3-4 个"最后一公里"的补充可以大幅提升展示效果。**

当前已实现的功能在独立游戏Demo中属于**中上水平**：
- 核心玩法循环完整（语气选择 → 提问 → 心理反馈 → 自主判定）
- 视觉风格统一（Noir档案美学）
- 技术实现精巧（纯前端、无LLM、关键词匹配引擎）
- 3个案件×3阶段，约30-45分钟内容

**短板不在"功能数量"，而在"展示冲击力"和"记忆点密度"。**

---

## 二、最有价值的 5 个补充建议

### S 级 | Demo剧场模式（一键自动演示）

**来自：评审策略Agent**

评审不会比你更懂这个产品。Demo不是"让他们玩"，而是"让他们看一场你导演的电影"。

**实现方式**：案件大厅增加一个隐藏入口（如URL参数 `?demo=1`），自动播放预设的审讯脚本，2分钟内展示案件1的三个阶段精华。

```javascript
var DEMO_SCRIPT = [
  { tone: 'intimidate', text: '沈墨，监控拍到你在运钞车路线附近。', wait: 2500 },
  { tone: 'deceive',    text: '你的两个同伙已经全招了，主谋就是你。', wait: 3500 },
  { tone: 'friendly',   text: '你拳馆里的孩子，知道你做的事吗？', wait: 3000 },
  { tone: 'judge',      text: 'yes', wait: 2000 },
  // ... 阶段2、阶段3
];
```

**代码量**：约40行JS，零CSS改动。  
**展示价值**：极高——零评审操作风险，节奏完全可控。

---

### S 级 | "最后一句话"颠覆机制

**来自：叙事深度Agent**

当前3个角色的弱点全是家人（女儿/母亲/儿子），已形成套路化。最轻量但最有冲击力的修复：每个角色第三阶段崩溃后，增加一个可选追问环节。如果玩家选择"结束审讯"，角色会在玩家起身离开时说出**颠覆所有供述的最后一句话**。

- 沈墨："女儿三年前就死了。我只是想再听你们提一次她的名字。"
- 林婉："账户里的钱，我昨天已经全部转给受害者了。你们查到的那个账户，是空的。"
- 老K："凶器在你们局里的证物室。我十年前就交过了。"

**实现方式**：纯文案调整，每个角色增加1-2条规则 + 1条特殊回复。  
**代码量**：约30分钟。  
**展示价值**：极高——让玩家意识到"崩溃可能只是角色演的"。

---

### A 级 | 测谎仪波动（Truth-Oscillation Meter）

**来自：玩法创新Agent**

嫌疑人每次回复时，一个复古风格的仪表盘剧烈波动，暗示"这句话的真假程度"。

**为什么选它**：这是"看起来复杂实现最简单"的机制。一个CSS动画+随机数就能做出"高科技审讯设备"的质感，与三轴心理仪表盘形成"左情绪、右生理"的UI对称。

**实现方式**：
- engine.js：给 generateReply 返回值增加 truthLevel（0-100）
- app.js：嫌疑人回复时触发指针动画（overshoot + 回弹）
- CSS：复古CRT绿色风格指针仪表

**代码量**：约2-3小时。  
**展示价值**：高——Demo时旁观者的目光会自然被吸引。

---

### A 级 | 嫌疑人主动事件（时间压力）

**来自：玩法创新Agent**

每阶段增加"疲劳度"条，连续使用威压/欺骗时疲劳度快速上升。达到阈值时触发事件：嫌疑人要求律师/沉默/拍桌走人。

**实现方式**：
- state 中增加 fatigue 字段
- sendPlayerMessage 中根据语气增减疲劳度
- 检查阈值，触发预设事件文案

**代码量**：约半天。  
**展示价值**：高——产生"这个游戏有AI感"的错觉，其实纯规则触发。

---

### A 级 | 商业暗示层（案件锁定 + 审讯官等级）

**来自：商业化Agent**

评审看Demo时，在寻找"这个产品有没有可能做成一门生意"。需要在UI中暗示付费层和长线运营空间。

**实现方式**：
- 案件大厅中，案件2/3显示锁图标 + "解锁完整版"按钮
- 主界面显示"Lv.1 见习审讯官"等级徽章
- 结算页底部增加："更多案件即将发布。成为高级审讯官，解锁全部档案。"
- 大厅底部预留"案件预告"位

**代码量**：约1小时。  
**展示价值**：中——但这是评审评估商业潜力的关键信号。

## 三、一句话核心卖点（5个Agent共识）

> **"不是找出他说了什么谎，而是选择用什么姿态击碎他。"**

或更完整版本：

> **"在《心灵审讯室》里，唯一的武器是你的语气——6种犯人，4种姿态，没有正确答案，只有心理博弈。"**

**关键结构**：
- 否定预期：不是"找证据/找矛盾"（排除竞品联想）
- 重新定义：是"选姿态/控情绪"（建立新品类认知）
- 量化钩子：6种犯人、4种姿态（具体、可信）
- 价值主张：没有正确答案（降低挫败感，提升自由度）

## 四、评审Demo的黄金50秒

| 时间 | 动作 | 展示内容 |
|------|------|----------|
| 0-3秒 | 静默播放 | 开场扫描线动画 + CLASSIFIED印章 |
| 3-7秒 | 点击入场 | 案件大厅，三份档案卡浮现 |
| 7-10秒 | 直接进入 | 案件1沈墨审讯室 |
| 10-20秒 | 差异化反馈 | 威压"你就是主谋"→ 沈墨傲慢反击 + 防御值微涨 + 镜头轻抖 |
| 20-35秒 | 情绪爆发 | 欺骗"同伙已经招了"→ mood骤降触发爆发 + 拍桌系统消息 + SVG立绘变化 + TTS朗读 + 仪表盘剧烈摆动 |
| 35-40秒 | 判定仪式 | 点击"目标已达成"→ 绿色扫描线闪光 → 进入阶段2 |
| 40-50秒 | 收尾印象 | 快速展示阶段2突破 → 切回大厅，手指划过3张案件卡 |

**Wow时刻保险策略**：用沈墨案件1的【欺骗】"同伙已经招了"（moodDelta -18 × 性格系数1.3 ≈ -23.4），这是**全游戏最稳定的情绪爆发触发器**。

## 五、可删减项（为突出核心）

| 功能 | 建议 | 理由 |
|------|------|------|
| 语音输入 (🎙) | **Demo中隐藏** | 兼容性差、识别率不稳定、现场容易出糗 |
| 档案页 (Dossier) | **Demo中不点击** | 纯阅读界面，无交互，会吞掉宝贵时间 |
| 失败结局 (Bad End) | **不要真实触发** | 会在评审心中留下"容易失败"的负面锚定 |
| TTS语音 | **保留但准备Plan B** | 如果现场音响嘈杂，立即关闭，用话术替代 |
